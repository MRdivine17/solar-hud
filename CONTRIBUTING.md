# 🤝 Contributing to Solar Hub

Thanks for your interest in contributing to **Solar Hub**!
We welcome contributions that improve the project, fix bugs, or add useful features.

---

## 📌 Before You Start

* Make sure you understand the purpose of this project
* Check existing issues to avoid duplicates
* Follow the project structure and coding style

---

## 🛠️ How to Contribute

### 1. Fork the Repository

Click the **Fork** button on GitHub and clone your fork:

```bash
git clone https://github.com/your-username/solar-hub.git
```

---

### 2. Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

---

### 3. Make Your Changes

* Keep your code clean and readable
* Follow Lua/FiveM best practices
* Test your changes before submitting

---

### 4. Commit Your Changes

```bash
git commit -m "Added: short description of your feature"
```

---

### 5. Push to GitHub

```bash
git push origin feature/your-feature-name
```

---

### 6. Create a Pull Request

* Go to your fork on GitHub
* Click **New Pull Request**
* Clearly explain what you added or fixed

---

## 📋 Contribution Rules

* ✔ Code must be properly tested
* ✔ No unnecessary or spam changes
* ✔ Keep commits clean and meaningful
* ✔ Respect project structure

---

## ❌ What is NOT Allowed

* Uploading stolen or copied code
* Breaking existing functionality
* Adding malicious or harmful scripts
* Re-uploading this project elsewhere (see license)

---

## 🧠 Coding Guidelines

* Use clear variable names
* Avoid unnecessary complexity
* Comment important logic
* Keep performance in mind (especially for FiveM servers)

---

## 🐞 Reporting Bugs

When reporting a bug, include:

* Clear description
* Steps to reproduce
* Screenshots (if possible)
* Expected vs actual behavior

---

## 💡 Suggesting Features

* Explain the idea clearly
* Describe how it improves the script
* Keep suggestions realistic

---

## 🚀 Final Note
